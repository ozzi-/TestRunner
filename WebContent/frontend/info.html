Info

Test Runner allows you to execute automated tests in a comfortable manner while persisting the results.
Test Runner consists of two parts, the API which is used to run / monitor tests and retrieve test results, the other being the lightweight UI which uses XHR requests on said API.
Further information can be found online on the GitHub repository.

Using as home.

Tests

Tests are a collection of 1 to n coherent tasks.
A task points to a executable which is then run in the context of the test.
Tests are defined in a JSON file ending with .test under "tests".
Example of a test:

	{
	  "settings": {
	  },
	  "test": {
	    "description": "test_windows",
	    "tasks": [{
	      "name": "task1",
	      "path": "script1.bat",
	      "timeout": 1
	      },{
	      "name": "task2",
	      "path": "script2.bat",
	      "timeout": 5
	      }]
	  }
	}


Groups

Groups allow the grouping of multiple tests into one and are defined under "groups". 
{
	"description": "test_windows",
	"tests": [{
		"test": "windows",
		"name": "1/2"
	},{
		"test": "windows2",
		"name": "2/2"
	}]
}


Results

Results are persisted as a JSON file.
Results are saved under "results\<TEST_NAME>" or in the case of grouped tests "results\groups\<GROUP_NAME>"
